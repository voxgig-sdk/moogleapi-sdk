# moogleapi-mcp

MCP server exposing the Moogleapi SDK as tools, built on the
[official Go MCP SDK](https://github.com/modelcontextprotocol/go-sdk)
and the sibling Go SDK at `../go`.

## Tools

| Tool | Args | Returns |
|------|------|---------|
| `moogleapi_list` | `entity`, optional `query` map | First page of records as JSON |
| `moogleapi_load` | `entity`, `query` (e.g. `{id:N}`) | Single record as JSON |

JSON schemas for both tools are emitted by the SDK from the `Args`
struct's `json` / `jsonschema` tags — no schema is hand-written.

## Build

```sh
go build -o moogleapi-mcp ./...
```

## Run

```sh
# stdio transport — for spawnable agent installs (default).
./moogleapi-mcp

# streamable HTTP transport — share one running server between agents.
./moogleapi-mcp -transport http -addr :8080
```

## Install for Claude Code

```sh
claude mcp add --scope user moogleapi \
  -- /absolute/path/to/moogleapi-mcp -transport stdio
```

After install, restart Claude Code; the `moogleapi_list` and
`moogleapi_load` tools become available in new sessions.

## Smoke test via HTTP

```sh
./moogleapi-mcp -transport http -addr :18080 &

# initialize, grab the session id
curl -sN -X POST http://localhost:18080 \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json, text/event-stream' \
  -D headers \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-06-18","capabilities":{},"clientInfo":{"name":"smoke","version":"0"}}}'

SESSION=$(awk '/Mcp-Session-Id/ {print $2}' headers | tr -d '\r')

curl -sN -X POST http://localhost:18080 \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json, text/event-stream' \
  -H "Mcp-Session-Id: $SESSION" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"moogleapi_load","arguments":{"entity":"moogle_api_web_features_characters_get_all_get_all_character","query":{"id":1}}}}'
```
